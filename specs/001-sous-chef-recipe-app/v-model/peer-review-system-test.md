### PRF-STP-001 — Non-Standard Technique Label: "Interface Contract Testing"

| Field              | Value                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Severity**       | Major                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| **Location**       | ISO 29119 Test Techniques section; STP-001-A, STP-002-A, STP-004-A, STP-005-A, STP-006-B, STP-007-A, STP-008-A, STP-009-A, STP-010-A, STP-011-A, STP-012-A, STP-013-A, STP-015-A, STP-016-A, STP-017-A, STP-019-A, STP-020-A                                                                                                                                                                                                                                                                                                                                                                |
| **Description**    | "Interface Contract Testing" is not a named technique in the ISO 29119-4 catalogue. ISO 29119-4 defines: Equivalence Partitioning, Boundary Value Analysis, Decision Table Testing, Cause-Effect Graphing, State Transition Testing, Pairwise Testing, Error Guessing, Exploratory Testing, and Specification-Based Testing (as a category). Using a non-standard label means the artifact cannot be validated against the standard's technique applicability criteria, and automated tooling that checks for ISO 29119-4 named techniques will flag these 17 test cases as non-conformant. |
| **Recommendation** | Remap "Interface Contract Testing" to the closest ISO 29119-4 named technique. For API endpoint contract verification, "Specification-Based Testing" (ISO 29119-4 §5) is the appropriate category, with "Equivalence Partitioning" as the primary technique for input/output class validation. Update the technique declaration in each affected STP and in the ISO 29119 Test Techniques section header.                                                                                                                                                                                   |

| Field              | Value                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| ------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Severity**       | Major                                                                                                                                                                                                                                                                                                                                                                                                                                                           |

| **Description**    | "Fault Injection" is a reliability engineering and hardware testing technique (per IEC 61508 / DO-178C) and is not listed in the ISO 29119-4 technique catalogue. Using it as an ISO 29119 technique label creates a standards compliance gap. The intent of these test cases — exercising failure propagation, dependency degradation, and error-path behavior — is valid and important, but the technique name must be drawn from the ISO 29119-4 vocabulary. |

Update the technique declaration in each affected STP and in the header section.                                                                                                                 |

### PRF-STP-003 — Archive State Machine Lacks Dedicated State Transition Test Case

| Field              | Value                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Severity**       | Major                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| **Location**       | SYS-009 / SYS-010 section; STP-009-A, STP-009-B, STP-010-A, STP-010-B                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| **Description**    | The system design defines a five-state archive lifecycle: `ARCHIVE_PENDING` → `ARCHIVE_IN_FLIGHT` → `NORMAL` (success path) and `ARCHIVE_IN_FLIGHT` → `ARCHIVE_DEAD_LETTER` → `ARCHIVE_IN_FLIGHT` (DLQ replay path). STP-007-C and STP-013-C correctly apply State Transition Testing to the photo and erasure lifecycles respectively. However, no STP for SYS-009 or SYS-010 applies State Transition Testing to the archive state machine. STP-010-B3 covers DLQ replay as a fault scenario but does not enumerate all valid/invalid transitions or terminal-state re-entry guards. This is an asymmetric coverage gap given that the archive lifecycle is the most operationally complex state machine in the system. |
| **Recommendation** | Add `STP-010-C` with technique "State Transition Testing" targeting the archive lifecycle. Include scenarios for: (1) valid path `ARCHIVE_PENDING` → `ARCHIVE_IN_FLIGHT` → `NORMAL`; (2) failure path `ARCHIVE_IN_FLIGHT` → `ARCHIVE_DEAD_LETTER`; (3) DLQ replay `ARCHIVE_DEAD_LETTER` → `ARCHIVE_IN_FLIGHT`; (4) invalid transition attempt on an already-`NORMAL` archived version; (5) terminal-state re-entry guard preventing duplicate archival.                                                                                                                                                                                                                                                                   |

### PRF-STP-004 — STS-003-B2 Compound Precondition Violates Single-Condition Principle

| Field              | Value                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Severity**       | Minor                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| **Location**       | STS-003-B2                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| **Description**    | The Given clause of STS-003-B2 contains a compound precondition joined by "AND": "a user has cloned an imported recipe (cloned recipe exists in user's library) AND the cloned recipe has had only metadata edits (title, tags) and photo edits applied — no substantive content edits to ingredients/instructions." ISO 29119-4 Boundary Value Analysis scenarios should establish a single, unambiguous boundary condition per scenario. The compound precondition conflates two independent setup states (clone existence and edit history), making the scenario harder to automate, harder to diagnose on failure, and potentially masking which condition is the actual boundary under test. |
| **Recommendation** | Split STS-003-B2 into two atomic scenarios: (a) a clone exists with zero edits applied — verify substantive classification is false; (b) a clone exists with only metadata/photo edits applied — verify substantive classification remains false. Each scenario should have a single, independently verifiable Given clause.                                                                                                                                                                                                                                                                                                                                                                      |

### PRF-STP-005 — STP-005-B2 Mixes Functional Boundary and Non-Functional Performance Testing

| Field              | Value                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Severity**       | Minor                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| **Location**       | STP-005-B (Description and STS-005-B2)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| **Description**    | STP-005-B is declared as "Search Parameter and Latency Boundaries" with technique Boundary Value Analysis. STS-005-B1 correctly tests pagination boundary values (functional). STS-005-B2 tests wall-clock response time and p95 latency under sustained load — this is a non-functional performance test, not a boundary value test. ISO 29119-4 distinguishes functional test design techniques (BVA, EP, etc.) from non-functional testing (performance, load, stress), which are governed by ISO 29119-10. Mixing them in a single STP under a functional technique label creates ambiguity about execution environment, tooling requirements, and pass/fail criteria authority. |
| **Recommendation** | Extract STS-005-B2 into a separate test case `STP-005-C` with technique "Performance Testing" (ISO 29119-10) and explicit non-functional test conditions: dataset size, load profile, measurement method (wall-clock vs. instrumented p95), and environment requirements (indexed schema, representative data volume). Retain STP-005-B as a pure BVA test case covering only pagination/filter boundary values.                                                                                                                                                                                                                                                                     |

### PRF-STP-006 — Decision Table Testing Absent for Multi-Condition Visibility Policy

| Field              | Value                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| ------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Severity**       | Minor                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| **Location**       | STP-003-A, STP-003-B                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| **Description**    | SYS-003 (Visibility and Source Policy Engine) enforces a multi-condition policy matrix: subscription tier (premium / free / lapsed) × source type (standard / public-source / paid-source / physical-copy) × edit state (substantive / non-substantive) → visibility outcome (allowed / denied / ruleId). STP-003-A uses Equivalence Partitioning and STP-003-B uses Boundary Value Analysis, which cover representative classes and boundary conditions. However, the combinatorial interaction between tier, source type, and edit state is not systematically exercised. ISO 29119-4 §5.4 Decision Table Testing is specifically designed for this pattern and would expose untested condition combinations (e.g., lapsed-tier + physical-copy + substantive-edit). |
| **Recommendation** | Add `STP-003-C` with technique "Decision Table Testing" that enumerates the full condition matrix for the visibility policy engine. At minimum, cover the cross-product of the three primary conditions (tier × source × edit state) for the `private` visibility transition, producing a decision table with explicit expected outcomes for each combination.                                                                                                                                                                                                                                                                                                                                                                                                         |

### PRF-STP-007 — Error Guessing Underutilized Across High-Risk Components

| Field              | Value                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| ------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Severity**       | Minor                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| **Location**       | ISO 29119 Test Techniques section; STP-001 through STP-020 (Error Guessing applied only in STP-008-C)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| **Description**    | Error Guessing (ISO 29119-4 §5.9) is declared as a named technique in the header but is applied in only one test case (STP-008-C) across 43 test procedures. Several high-risk components have well-known failure modes that are not covered by formal partitions or boundary analysis and would benefit from Error Guessing test cases: STP-001 (JWT clock skew, token replay, JWKS cache poisoning), STP-006 (partial multipart upload, concurrent confirm calls for the same object key), STP-013 (erasure job submitted during active session, concurrent erasure + recipe create), STP-010 (message visibility timeout expiry mid-processing). Declaring a technique in the header without applying it substantively is a conformance gap. |
| **Recommendation** | Add Error Guessing test cases to at least three additional high-risk components. Prioritize: (1) STP-001-C for JWT edge cases (clock skew, replay, JWKS unavailability during cache miss); (2) STP-006-C for concurrent upload/confirm race conditions; (3) STP-013-D for concurrent erasure + active-session edge cases. Each new STP should document the risk rationale in its Description field.                                                                                                                                                                                                                                                                                                                                             |

### PRF-STP-008 — STS-008-B1 Setup Requires 12 Sequential Saves (Implicit Ordering Dependency)

| Field              | Value                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Severity**       | Observation                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| **Location**       | STS-008-B1                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| **Description**    | STS-008-B1 specifies "12 successful saves for the same recipe resulting in versions 1..12" as its Given precondition. While this is a valid boundary condition for the 10-version retention rule, the scenario implicitly requires 12 sequential write operations to establish the precondition. This is not a shared-state dependency between scenarios, but it does mean the scenario cannot be executed with a simple fixture — it requires either a seeded dataset with 12 pre-existing versions or a setup harness that performs 12 saves. The scenario description does not specify whether seeded data or live saves are acceptable, which may cause inconsistent test execution across environments. |
| **Recommendation** | Clarify the Given clause to explicitly permit seeded test data: "Given a recipe with 12 pre-seeded version rows (versions 1..12) established via deterministic seed fixtures." This aligns with the project's stated use of deterministic seeded IDs (per STP-020-B1) and removes ambiguity about whether the 12 saves must be performed live during test execution.                                                                                                                                                                                                                                                                                                                                         |
