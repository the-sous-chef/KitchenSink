### PRF-UTP-001 — Addendum IDs Break UTP Schema (MOD Traceability Lost)

| Field              | Value                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Severity**       | Critical                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| **Location**       | Peer-Review Remediation Addendum — UTP-094-A, UTP-095-A, UTP-096-A, UTP-097-A, UTP-098-A, UTP-099-A                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| **Description**    | The six addendum test cases use IDs `UTP-094` through `UTP-099`, which do not follow the `UTP-{MOD_NNN}-{X}` schema defined in the document's own ID Schema section. The schema encodes MOD ancestry directly in the identifier (e.g., `UTP-001-D` for a fourth case on MOD-001). IDs `UTP-094` through `UTP-099` have no corresponding `MOD-094` through `MOD-099` in `module-design.md`, breaking the deterministic traceability chain that `validate-module-coverage.sh` and Matrix D rely on. The coverage summary counts these 6 cases in the total of 99 but they cannot be correctly linked to their parent modules by automated tooling. |
| **Recommendation** | Renumber the addendum cases using the correct schema: UTP-094-A → UTP-001-D (Error Guessing for MOD-001), UTP-095-A → UTP-005-D (Error Guessing for MOD-005), UTP-096-A → UTP-013-D (Error Guessing for MOD-013), UTP-097-A → UTP-024-D (Error Guessing for MOD-024), UTP-098-A → UTP-001-E (State Transition for MOD-001), UTP-099-A → UTP-024-E (State Transition for MOD-024). Update the Coverage Summary table accordingly.                                                                                                                                                                                                                 |

| ------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Severity**       | Major                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |

Five modules with documented State Machine Views in `module-design.md` have no State Transition test case: MOD-013 (`pending_processing → ready \| failed`), MOD-022 (`requested → db_done → storage_done → completed \| failed`), MOD-026 (photo widget `idle → presigning → uploading → confirming → processing → ready \| failed`; conflict resolver `clean → editing → conflict_detected → resolved`), MOD-027 (auth state `bootstrapping → authenticated \| needs_login`), and MOD-033 (app lifecycle `bootstrapping → listening → draining → stopped`). ### PRF-UTP-003 — Incorrect "None — module is self-contained" Mock Registry for MOD-002 and MOD-003

| ------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Severity**       | Major                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| **Location**       | UTP-002-B, UTP-003-B                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| **Description**    | UTP-002-B (action/tier partitions) and UTP-003-B (command-kind partitions) both declare `None — module is self-contained` in their Dependency & Mock Registry. However, MOD-002's Algorithmic View calls `repository.loadByKind` to fetch the resource, and MOD-003's Algorithmic View delegates to MOD-005 (validate), MOD-004 (execute), and MOD-010 (search). Executing these test cases without mocking those dependencies would invoke live repository or service logic, violating the strict isolation principle stated in the document's overview. Add a Dependency & Mock Registry table to UTP-003-B stubbing MOD-005 validator, MOD-004 command service, and MOD-010 search service.                                                                                                                                                                                                                                                                                                                                                                                                                                  |

### PRF-UTP-007 — UTP-008-B Boundary Description Misidentifies Minimum Boundary

| Field              | Value                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Severity**       | Minor                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| **Location**       | UTP-008-B, UTS-008-B1                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| **Description**    | UTP-008-B is titled "quantity boundaries" and its description states "boundary-adjacent values passed to normalizer." However, UTS-008-B1 uses `quantity=0`, which is the exact minimum boundary (the `@Min(0)` constraint from MOD-005's `IngredientItem`), not a boundary-adjacent value. ISO 29119-4 BVA requires probing min-1 (e.g., `quantity=-0.001`), min (`quantity=0`), a mid value, max, and max+1. The scenario only covers the min point and does not probe the min-1 (below-minimum) rejection path, leaving the lower boundary guard untested. |
| **Recommendation** | Add a second scenario UTS-008-B2 with `quantity=-0.001` asserting that the normalizer or resolver rejects or handles the sub-zero value. Rename the description to accurately state "minimum boundary value" rather than "boundary-adjacent values."                                                                                                                                                                                                                                                                                                          |
