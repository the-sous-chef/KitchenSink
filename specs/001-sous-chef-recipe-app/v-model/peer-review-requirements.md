### PRF-REQ-001 — Umbrella IDs Duplicate Sub-ID Obligations (Atomicity)

| Field              | Value                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| ------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Severity**       | Major                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| **Location**       | REQ-035, REQ-038, REQ-049                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| **Description**    | Three umbrella IDs (REQ-035, REQ-038, REQ-049) remain in the requirements table alongside their fully decomposed sub-IDs (REQ-035a/b, REQ-038a/b/c, REQ-049a/b). Each umbrella row contains compound obligations that are already expressed atomically in the sub-IDs. This creates duplicate, overlapping requirements: a test suite could satisfy the umbrella without exercising all sub-IDs, or vice versa. INCOSE requires each requirement to be independently verifiable with a single verification event; umbrella rows that restate compound obligations violate this principle. |
| **Recommendation** | Convert umbrella rows to non-normative index entries (e.g., move them to a comment or a separate "decomposition map" table outside the normative requirements table), or annotate them explicitly as `[NON-NORMATIVE — see sub-IDs]` so that verification tooling skips them. The sub-IDs (REQ-035a/b, REQ-038a/b/c, REQ-049a/b) are the authoritative normative requirements.                                                                                                                                                                                                            |

### PRF-REQ-006 — REQ-NF-008 Uses Ambiguous Negation Construct (Unambiguity)

| Field              | Value                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Severity**       | Major                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| **Location**       | REQ-NF-008                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| **Description**    | REQ-NF-008 reads: "No UI component in this feature MUST convey state using color alone." The construction "No X MUST Y" is grammatically ambiguous in RFC 2119 / INCOSE contexts: it can be parsed as either (a) "No UI component is required to convey state using color alone" (i.e., color-only is optional) or (b) "No UI component is permitted to convey state using color alone" (i.e., color-only is prohibited). The intended meaning is clearly (b), but the phrasing does not unambiguously express it. |
| **Recommendation** | Rewrite using an unambiguous prohibition: "The system MUST NOT convey UI state using color as the sole differentiator; every state indicator MUST also include an icon or text label." This eliminates the "No X MUST" anti-pattern.                                                                                                                                                                                                                                                                               |

### PRF-REQ-007 — REQ-CN-003 Uses Non-Standard Modal "MUST NEVER" (Unambiguity)

| Field              | Value                                                                                                                                                                                                                                                                                                                                        |
| ------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Severity**       | Major                                                                                                                                                                                                                                                                                                                                        |
| **Location**       | REQ-CN-003                                                                                                                                                                                                                                                                                                                                   |
| **Description**    | REQ-CN-003 uses "MUST NEVER," which is not a defined modal keyword in RFC 2119 or the INCOSE Guide for Writing Requirements. The standard modals are MUST, MUST NOT, SHALL, SHALL NOT, SHOULD, SHOULD NOT, MAY, and NEED NOT. "MUST NEVER" introduces an undefined term that could be interpreted differently by different readers or tools. |
| **Recommendation** | Replace "MUST NEVER be settable" with "MUST NOT be settable." The prohibition is already absolute due to the "regardless of subscription tier or edit history" clause; "MUST NOT" is sufficient and unambiguous.                                                                                                                             |
