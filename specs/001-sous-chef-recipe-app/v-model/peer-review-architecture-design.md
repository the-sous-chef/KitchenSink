## Findings

### PRF-ARCH-001 — Missing Architectural Decisions and Rationale Section

| Field              | Value                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| ------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Severity**       | Major                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| **Location**       | Overall document structure                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| **Description**    | IEEE 42010 §7 requires architectural decisions and their rationale to be explicitly documented. The artifact embeds rationale fragments in the Overview paragraph and in individual table cells (notably ARCH-033's Parent System Components cell), but there is no dedicated "Architectural Decisions" section. Key decisions with significant design impact are undocumented: (1) NestJS as the API framework, (2) SQS for the version-archive queue rather than EventBridge or direct Lambda invocation, (3) optimistic concurrency (ARCH-016) over pessimistic locking, (4) Lambda for photo processing (ARCH-014) rather than a container sidecar, (5) Drizzle ORM over raw SQL or another ORM. Without explicit rationale, reviewers and future maintainers cannot evaluate whether alternatives were considered or whether the decisions remain valid as constraints change. |
| **Recommendation** | Add an "Architectural Decisions" section (or an ADR table) before the Coverage Summary. For each significant decision, record: Decision ID, Decision statement, Alternatives considered, Rationale for selection, and Constraints/assumptions that could invalidate the decision. At minimum cover the five decisions listed above.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |

### PRF-ARCH-002 — No Reverse SYS→ARCH Traceability Table

| Field              | Value                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| ------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Severity**       | Major                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| **Location**       | Logical View / Coverage Summary                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| **Description**    | The artifact provides forward traceability (each ARCH module lists its parent SYS IDs) but no reverse lookup table showing which ARCH modules implement each SYS component. This makes it impossible to verify decomposition completeness without manually scanning all 33 rows. For example, SYS-002 (Recipe Command Service) is implemented by ARCH-003, ARCH-004, ARCH-005, and ARCH-028 — but this is only discoverable by inspection. The Coverage Summary claims "20/20 SYS covered" but provides no evidence of the mapping. Under-decomposed SYS components (a SYS mapped to only one thin ARCH module) or over-decomposed ones cannot be identified without the reverse table. |
| **Recommendation** | Add a "SYS→ARCH Reverse Traceability" table after the Logical View (or in the Coverage Summary section) with columns: SYS ID, SYS Name, Implementing ARCH IDs, ARCH Count. This enables reviewers to spot SYS components with suspiciously few or many ARCH modules and validates the "100% Forward Coverage" claim.                                                                                                                                                                                                                                                                                                                                                                    |

### PRF-ARCH-003 — Physical View Deployment Target Ambiguity

| Field              | Value                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| ------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Severity**       | Minor                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| **Location**       | Physical View — Runtime/Infrastructure Topology, Runtime Allocation Matrix                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| **Description**    | The Physical View consistently uses "ECS Fargate or App Runner" (with "or") throughout the topology description and the Runtime Allocation Matrix. IEEE 42010 expects the Physical View to describe the actual deployment topology, not a set of alternatives. The two options have meaningfully different operational characteristics: ECS Fargate requires task definitions, service auto-scaling, and ALB configuration; App Runner abstracts these but has different VPC connector semantics, concurrency limits, and cold-start behavior. Leaving this unresolved means the Physical View cannot be used to derive infrastructure-as-code (ARCH-032) or to evaluate deployment-related hazards. |
| **Recommendation** | Commit to one runtime target in the Physical View. If the decision is genuinely deferred, document it as an open architectural decision (see PRF-ARCH-001) with the criteria that will drive the selection (e.g., team operational familiarity, VPC connector requirements, cost model). Replace "ECS Fargate or App Runner" with the chosen option throughout the Physical View and Runtime Allocation Matrix.                                                                                                                                                                                                                                                                                      |

### PRF-ARCH-004 — ARCH-003 Interface Contract Incomplete for Guard-Level Rejections

| Field              | Value                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Severity**       | Minor                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| **Location**       | Interface View — ARCH-003: Recipe HTTP Controller                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| **Description**    | The ARCH-003 interface table lists a single Exception row: "mapped via ARCH-028" — delegating all error documentation to the error mapper. However, ARCH-003 is the entry point for guard-level rejections (ARCH-001 JWT verification failures, ARCH-002 ownership/tier denials) that occur before the command service is invoked. The interface contract does not enumerate the HTTP status codes the controller can emit (401 from ARCH-001, 403 from ARCH-002, 404 for unknown recipe IDs in path parameters, 405 for unsupported methods). Integration test authors targeting ARCH-003 cannot derive complete test cases from this interface definition alone. |
| **Recommendation** | Expand the ARCH-003 interface table to enumerate the guard-level exception paths explicitly: add rows for INVALID_TOKEN (401, from ARCH-001), FORBIDDEN_OWNER / FORBIDDEN_TIER (403, from ARCH-002), RECIPE_NOT_FOUND (404, path param validation), and METHOD_NOT_ALLOWED (405). Note that these are surfaced via ARCH-028 but originate at the controller boundary.                                                                                                                                                                                                                                                                                              |

### PRF-ARCH-005 — ARCH-028 Interface Does Not Enumerate Handled Domain Error Codes

| Field              | Value                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| ------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Severity**       | Minor                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| **Location**       | Interface View — ARCH-028: API Error Mapper                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| **Description**    | The ARCH-028 interface table describes its input as "any Error subclass" with the note "Domain errors carry `code` and HTTP status hint." This is insufficient to verify completeness of error mapping coverage. The artifact defines specific domain error codes across other modules (INVALID_TOKEN, FORBIDDEN_OWNER, FORBIDDEN_TIER, CONCURRENCY_CONFLICT, POLICY_DENIED, VALIDATION_FAILED, INGREDIENT_NOT_FOUND, UNIT_INCONVERTIBLE, SEARCH_TIMEOUT, UPLOAD_QUOTA_EXCEEDED, UPLOAD_INVALID, UPLOAD_NOT_FOUND, PROCESSING_FAILED, VERSION_WRITE_FAILED, QUEUE_SEND_FAILED, ARCHIVE_WRITE_FAILED, RECONCILER_LOCK_LOST, NOT_FOUND, PULL_LOCK_HELD, ERASURE_FAILED, STORAGE_PURGE_PARTIAL, DB_UNAVAILABLE, DB_CONSTRAINT, S3_ERROR, S3_THROTTLED, CONFIG_INVALID, DI_RESOLUTION_FAILED) but ARCH-028 does not reference them. | **Location**       | Logical View — ARCH-033: NestJS Module Wiring                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |

### PRF-ARCH-007 — "+1 Scenarios" View Not Explicitly Identified

| Field              | Value                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Severity**       | Minor                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| **Location**       | Document structure — view labeling                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| **Description**    | Kruchten 4+1 defines exactly five views: Logical, Process, Physical, Development, and "+1 Scenarios" (the unifying use-case view that drives and validates the other four). The artifact presents six named views: Logical, Process, Physical, Development, Interface, and Data Flow. Neither "Interface View" nor "Data Flow View" is labeled as the "+1 Scenarios" view. The Overview states "five critical end-to-end flows" in the Process View, which could serve as the +1 Scenarios, but the Process View is labeled as "Dynamic Behavior" — a different Kruchten slot. The standard requires the +1 view to be explicitly identified so reviewers can verify that the scenarios exercise all four primary views. |
| **Recommendation** | Explicitly identify which view serves as the "+1 Scenarios" view in the document header or a "View Mapping" table. If the five Process View sequence diagrams serve this role, label the Process View section as "Process View (+1 Scenarios)" and add a sentence explaining how each scenario exercises the Logical, Physical, and Development views. If the Data Flow chains serve this role instead, relabel accordingly.                                                                                                                                                                                                                                                                                             |

| ------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Severity**       | Observation                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| **Location**       | Coverage Summary                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| **Description**    | The Coverage Summary reports "Cross-Cutting Modules: 1 (ARCH-033)." This metric counts only modules tagged `[CROSS-CUTTING]`, but ARCH-029 (Config Loader), ARCH-030 (Telemetry & Logger), ARCH-031 (Archive Backlog Alarm), and ARCH-032 (CI & Test Governance Harness) are all Utility-type modules that serve cross-cutting concerns across multiple SYS components. ### PRF-ARCH-010 — Interface View Not a Named Kruchten 4+1 View

| Field              | Value                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| ------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Severity**       | Observation                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| **Location**       | Interface View section header                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| **Description**    | The "Interface View — API Contracts (Kruchten 4+1)" section label implies this is a standard Kruchten 4+1 view. Kruchten's model has four primary views (Logical, Process, Physical, Development) plus the +1 Scenarios view — there is no "Interface View" in the standard. The interface contracts are valuable and required by IEEE 42010 §8 (architecture description elements), but labeling them as a Kruchten 4+1 view is inaccurate and creates the same ambiguity as PRF-ARCH-009 regarding which view serves as the "+1 Scenarios" slot. |
| **Recommendation** | Relabel the section as "Interface View — API Contracts (IEEE 42010 §8 Extension)" or "Module Interface Specifications." Add a note clarifying this section provides the interface contracts required by IEEE 42010 but is not one of the five Kruchten 4+1 views. Cross-reference it from the Logical View table to make the relationship explicit.                                                                                                                                                                                                |
