### PRF-ITP-001 — CDCT Missing for Mobile Consumer Boundary (ARCH-027 ↔ ARCH-003)

| **Description**    | ARCH-027 (Mobile UI) is a distinct consumer of ARCH-003 with different token handling (`react-native-auth0` + `expo-secure-store`), a different base URL, and mobile-specific conflict UI parity requirements. | ------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

| **Location**       | Module ARCH-017 (ITP-017-A through ITP-017-D) and ARCH-018                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| **Description**    | The SQS archive queue boundary between ARCH-017 (Archive Queue Producer) and ARCH-018 (Archive Worker Lambda) is a classic producer/consumer seam with an explicit message schema (`{ pendingArchiveId, recipeId, versionNumber }`). This boundary has Interface Contract, Fault Injection, Data Flow, and Concurrency tests but no CDCT. | **Description**    | ARCH-007 (Substantive Edit Detector) has only one test case (ITP-007-A, Interface Contract Testing). Add `ITP-007-C` (Concurrency & Race Condition Testing) verifying that concurrent calls from ARCH-004 with different `beforeRecipe`/`afterRecipe` pairs return isolated, non-interleaved results.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
