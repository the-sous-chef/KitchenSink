## Overview

This document presents the Failure Mode and Effects Analysis (FMEA) for Sous Chef - Recipe
Management Core. Every system component (`SYS-NNN`) from `system-design.md` is assessed for
potential failure modes across relevant operational states. Each hazard receives a unique
`HAZ-NNN` identifier and is linked to risk control measures (`REQ-NNN` / `SYS-NNN`), enabling
the traceability chain: Hazard → Mitigation → Requirement → Test Case (Matrix H).

## ID Schema

- **Hazard ID**: `HAZ-{NNN}` — 3-digit zero-padded, sequential (HAZ-001, HAZ-002, ...)
- **ID Lineage**: From `HAZ-001`, read the Mitigation column to find `REQ-NNN` / `SYS-NNN`.
  Consult `traceability-matrix.md` (Matrix H) for the full chain to verification test cases.

## Risk Matrix Definition

### Severity Scale

| Level        | Definition                                                                                                |
| ------------ | --------------------------------------------------------------------------------------------------------- |
| Catastrophic | Mass user data loss, full account compromise, regulatory breach (e.g., GDPR/CCPA), platform outage > 24h. |
| Critical     | Single-user data loss, privacy breach (PII exposure), unauthorized access to recipes/collections.         |
| Serious      | Permanent feature degradation, broken core workflow (recipe save/load), auth disruption < 24h.            |
| Moderate     | Temporary feature unavailability, slow UX, recoverable errors with user-visible impact.                   |
| Minor        | Cosmetic issues, log noise, non-blocking warnings.                                                        |

### Likelihood Scale

| Level      | Definition                                                   |
| ---------- | ------------------------------------------------------------ |
| Frequent   | Likely to occur often; continuously experienced              |
| Probable   | Will occur several times; expected to occur                  |
| Occasional | Likely to occur sometime; can reasonably be expected         |
| Remote     | Unlikely but possible; could occur in the life of the system |
| Improbable | So unlikely it can be assumed it will not occur              |

### Risk Matrix (Severity × Likelihood)

|                  | Frequent     | Probable     | Occasional   | Remote      | Improbable  |
| ---------------- | ------------ | ------------ | ------------ | ----------- | ----------- |
| **Catastrophic** | Unacceptable | Unacceptable | Unacceptable | Undesirable | Undesirable |
| **Critical**     | Unacceptable | Unacceptable | Undesirable  | Undesirable | Tolerable   |
| **Serious**      | Unacceptable | Undesirable  | Undesirable  | Tolerable   | Tolerable   |
| **Moderate**     | Undesirable  | Tolerable    | Tolerable    | Acceptable  | Acceptable  |
| **Minor**        | Tolerable    | Acceptable   | Acceptable   | Acceptable  | Acceptable  |

### Residual Risk Methodology

Per ISO 14971 §7.4, every hazard is re-evaluated **after** the documented risk-control
measures (Mitigation column) are assumed to be fully implemented and verified. The post-
mitigation parameters are recorded in three dedicated columns:

- **Residual Severity** — severity level after mitigation reduces or contains impact (may
  remain unchanged if controls are detective/recovery-only).
- **Residual Likelihood** — likelihood level after mitigation reduces frequency of the
  failure mode reaching its effect.
- **Residual Risk Level** — the risk-matrix cell at (Residual Severity × Residual
  Likelihood). This MUST equal the matrix lookup; narrative claims that bypass the matrix
  are not permitted.

**ALARP justification**: For Catastrophic hazards (HAZ-024, HAZ-033, HAZ-052), the lowest
achievable residual risk per the matrix is **Undesirable** (Catastrophic × Improbable).
These remain Undesirable after mitigation and are explicitly accepted under ALARP ("As Low
As Reasonably Practicable") with operator-confirmation safeguards, signed audit trails,
and runbook-driven recovery (see §ALARP Acceptance below).
