### PRF-SYS-007 — SYS-005 External Interface Error Handling Is an Implementation Note, Not a Contract

| Field              | Value                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Severity**       | Minor                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| **Location**       | Interface View (IEEE 1016 §5.3) — SYS-005 External Interface                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| **Description**    | The SYS-005 (Recipe Search and Filter Service) external interface states error handling as "400 for invalid filters, 401 for auth failure, latency guarded by indexed query plans." The phrase "latency guarded by indexed query plans" is an implementation note, not an error contract. The interface does not specify what response is returned when the query exceeds the REQ-NF-001 2.0-second SLO (e.g., 504 Gateway Timeout, 503 Service Unavailable, or a partial result), nor what happens when SYS-014 (Persistence Layer) is degraded. |
| **Recommendation** | Replace the implementation note with an explicit error contract: e.g., "Queries exceeding the 2.0-second SLO return HTTP 504 with `SEARCH_TIMEOUT` error code; SYS-014 unavailability surfaces as HTTP 503 with `PERSISTENCE_UNAVAILABLE`." This makes the interface independently testable against REQ-NF-001.                                                                                                                                                                                                                                   |

### PRF-SYS-013 — Operational States Not Cross-Referenced from Interface or Dependency Views

| Field              | Value                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| ------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Severity**       | Observation                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| **Location**       | Operational States section                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| **Description**    | The Operational States section defines 10 named states (NORMAL, PHOTO_PENDING, PHOTO_PROCESSING, PHOTO_FAILED, ARCHIVE_PENDING, ARCHIVE_IN_FLIGHT, ARCHIVE_DEAD_LETTER, ERASURE_REQUESTED, ERASURE_IN_PROGRESS, ERASURE_COMPLETED) with entry/exit conditions. For example, the SYS-006 external interface does not mention that a successful `confirm` call transitions the system from PHOTO_PENDING → PHOTO_PROCESSING, and the SYS-013 interface does not reference ERASURE_REQUESTED/ERASURE_IN_PROGRESS/ERASURE_COMPLETED. IEEE 1016 §5.3 interface specifications benefit from state-context annotations to make behavioral contracts unambiguous. |
| **Recommendation** | Add state-transition annotations to the relevant interface rows. For example, in the SYS-006 external interface Error Handling column, note: "Successful confirm transitions photo lifecycle PHOTO_PENDING → PHOTO_PROCESSING." In the SYS-013 external interface, note: "202 response corresponds to ERASURE_REQUESTED state; 410 response corresponds to ERASURE_COMPLETED state." This links the interface contracts to the operational state machine without duplicating the full state table.                                                                                                                                                                                                                                                   |
