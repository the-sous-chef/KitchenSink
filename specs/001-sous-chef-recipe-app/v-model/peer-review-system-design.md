### PRF-SYS-002 — `recipe_tags` Data-Ownership Component Column Lists Two Components

| Field              | Value                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Severity**       | Observation                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| **Location**       | Data Design View (IEEE 1016 §5.4) — `recipe_tags` row (line 155)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| **Description**    | The `recipe_tags` entity row lists its Component as `SYS-002 / SYS-005`. Every other row in the Data Design View identifies a single owning component (typically `SYS-014` for relational tables, `SYS-015` for object storage, `SYS-013` for the erasure ledger). Listing two components in the Component column blurs write-vs-read ownership: SYS-002 (Recipe Command Service) owns mutation, SYS-005 (Recipe Search and Filter Service) only reads/queries. IEEE 1016 §5.4 expects one accountable component per data entity for retention and access-control attribution; reader components are typically captured implicitly through their dependency on the persistence layer. |
| **Recommendation** | Set the Component column to `SYS-014` (consistent with all other PostgreSQL tables) since the table is owned by the persistence layer at storage level, and add a note in the Description column that SYS-002 owns write paths and SYS-005 consumes via read-only queries. Alternatively, set Component to `SYS-002` if the design intent is to attribute domain ownership to the command service; either choice is acceptable so long as it matches the convention used by `recipes`, `recipe_steps`, and other tables.                                                                                                                                                              |

### PRF-SYS-003 — REQ-NF-018 / REQ-NF-019 / REQ-NF-020 Ownership Overlap Between SYS-016, SYS-017, and SYS-018

| Field              | Value                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| ------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Severity**       | Observation                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| **Location**       | Decomposition View (IEEE 1016 §5.1) — SYS-016, SYS-017, SYS-018                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| **Description**    | `REQ-NF-018` is listed as a parent of both SYS-016 (Web Client) and SYS-018 (Configuration). `REQ-NF-019` is listed as a parent of both SYS-017 (Mobile Client) and SYS-018. `REQ-NF-020` is owned only by SYS-018. Many-to-many traceability is permitted by the ID Schema, but the responsibility split between "consumer of the configuration value" (SYS-016/SYS-017) and "provider of the typed configuration loader" (SYS-018) is not documented. Without this split, a downstream Matrix B test author cannot tell whether to verify these REQs against the client (consumer behavior) or against the configuration utility (loader behavior), and they may be redundantly tested or under-tested. The Coverage Summary correctly reports them as covered, but the verification ownership is ambiguous between three SYSes. |
| **Recommendation** | Add a brief sentence to each component's Description column (or to the Design Notes section) clarifying the split — e.g., SYS-018 owns "typed loading and default resolution," while SYS-016/SYS-017 own "correct consumption and runtime behavior of the resolved values." Alternatively, narrow the parent-requirement assignments so each REQ has one primary owner with the others marked as consumers in a separate "Consuming Components" column.                                                                                                                                                                                                                                                                                                                                                                            |

### PRF-SYS-004 — Coverage Summary Count Not Independently Verifiable From Decomposition View

| Field              | Value                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| ------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Severity**       | Observation                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| **Location**       | Coverage Summary (lines 187–198) and Decomposition View Parent Requirements column                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| **Description**    | The Coverage Summary asserts `Total Parent Requirements Covered = 129 / 129 (100%)` with the metric definition note clarifying that the denominator includes umbrella + atomic sub-IDs. The number is asserted but not derivable by hand from the Decomposition View alone — a reviewer must cross-check `requirements.md` to confirm the 129 denominator and trace each ID through the Parent Requirements columns. This is acceptable for an AI-generated artifact, but it relies entirely on `build-matrix.sh` (deterministic script) being run downstream to verify the count. If that script is not re-run after subsequent edits, the `129 / 129` figure may silently drift out of sync with the actual REQ count or the SYS coverage. |
| **Recommendation** | Add a footnote or commit-time check that the Coverage Summary number is regenerated by `build-matrix.sh` and not hand-edited. Optionally include a small "Coverage by Component" sub-table listing each SYS with its parent-requirement count, which would make the total visually checkable without external tools. No change required if Matrix B is regenerated as part of every artifact update.                                                                                                                                                                                                                                                                                                                                         |
